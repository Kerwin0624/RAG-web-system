# 默认参数完整说明及调优建议

## 分块参数

| 参数 | 默认值 | 范围 | 调优建议 |
|------|--------|------|---------|
| `CHUNK_SIZE` | 800 | 100–4000 | 中文约 400 字。短文档/精确 QA: 400–600; 长文档/摘要: 1000–2000 |
| `CHUNK_OVERLAP` | 120 | 0–1000 | 约 chunk_size 的 10%–20%。过大浪费存储，过小句子断裂 |

### 分隔符优先级

默认: `["\n\n", "\n", " ", ""]`

- 双换行先按段落切分
- 单换行再按行切分
- 空格按词切分（中文效果有限）
- 空字符串为最终回退

中文场景可考虑添加句号 `。` / 问号 `？` / 感叹号 `！` 作为分隔符。

## 检索参数

| 参数 | 默认值 | 范围 | 调优建议 |
|------|--------|------|---------|
| `SEARCH_TOP_K` | 4 | 1–20 | 精确问答: 2–4; 综合分析: 6–10 |
| `SEARCH_SCORE_THRESHOLD` | 0.25 | 0.0–1.0 | 降低→更多结果; 提高→更精准但可能漏答 |
| `VECTOR_WEIGHT` | 0.7 | 0.0–1.0 | 语义检索权重。关键词密集文档可降至 0.5 |
| `BM25_WEIGHT` | 0.3 | 0.0–1.0 | 与 vector_weight 互补，总和建议为 1.0 |

### 检索策略选择指南

```
需求 ──→ 推荐策略
│
├── 通用语义问答 ──→ similarity
├── 精确术语/代码 ──→ bm25
├── 中文复杂问题 ──→ hybrid (推荐)
├── 多角度覆盖 ──→ mmr
└── 高精度场景 ──→ hybrid + reranker
```

### Reranker 参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `RERANKER_MODEL` | BAAI/bge-reranker-v2-m3 | 支持中英文的轻量 CrossEncoder |
| fetch_k (代码内) | max(k*3, 20) | 重排序前先多取候选 |

- Reranker 会显著增加延迟（首次加载模型 ~10s，之后每次 ~100ms–1s）
- 建议仅在 `hybrid` 策略下启用，效果最佳

## 生成参数

| 参数 | 默认值 | 范围 | 调优建议 |
|------|--------|------|---------|
| `DEFAULT_TEMPERATURE` | 0.2 | 0.0–2.0 | RAG 推荐 0.0–0.3 以减少幻觉。创意任务可提高到 0.7+ |
| `DEFAULT_MAX_TOKENS` | 512 | 32–4096 | 简短回答: 256; 详细分析: 1024–2048 |
| `DEFAULT_TOP_P` | 1.0 | 0.1–1.0 | 配合 temperature 使用。一般保持 1.0 |

### Temperature vs Top-P 组合建议

| 场景 | Temperature | Top-P | 说明 |
|------|------------|-------|------|
| 事实性 QA | 0.0–0.2 | 1.0 | 最忠实上下文 |
| 一般问答 | 0.3–0.5 | 0.9 | 平衡忠实与流畅 |
| 摘要生成 | 0.5–0.7 | 0.95 | 允许适度改写 |
| 创意写作 | 0.8–1.2 | 0.9 | 更多随机性 |

## Pipeline 缓存

| 参数 | 默认值 | 位置 |
|------|--------|------|
| `cache_size` | 256 | `RAGPipeline.__init__()` 构造参数 |

- 缓存键: 所有查询参数的 MD5
- 相同问题+相同参数 = 缓存命中
- 修改任何参数即触发新查询
- 可通过重启服务清空缓存

---

## 典型调优流程

1. **先用默认参数**跑基线评测
2. 观察 RAGAS 指标：
   - `context_precision` 低 → 增大 `TOP_K`，启用 Reranker
   - `context_recall` 低 → 减小 `CHUNK_SIZE`，尝试 `hybrid` 检索
   - `faithfulness` 低 → 降低 `TEMPERATURE`
   - `answer_relevancy` 低 → 调整 Prompt 或换更强模型
3. 每次只调一个参数，对比评测结果
4. 使用"标准答案评测" Tab 的参数快照功能记录每次实验
